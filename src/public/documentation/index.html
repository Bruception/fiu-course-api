Combined Queries

                

                    Let's say we are trying to find all courses whose subject starts with J, code starts with 1, and is 5 units.
                    Here is an example of how you might query for this.    
                

                
GET /api?subject=J&code=1&units=5

                

            

            

                
Multi-value Queries

                

                    Let's say we are trying to find all labs whose subject starts with AST or COP.
                    Here is an example of how you might query for this.    
                

                
GET /api?subject=AST&subject=COP&isLab

                

            

            

                
Data Serialization

                

                    You can specify how the returned data is serialized with the format parameter.
                    Currently this API only supports the following serialization formats:
                    
    
                        
  • JSON
    • 
                        
  • XML
    • 
                        
  • YAML
    • 
                        
  • Protocol Buffers
    • 
                    

                    To use protocol buffers, specify the format as "protobuf".
                    Here is an example of how you might query your data in YAML.
                

                
GET /api?format=yaml

            

            

                
Keyword Search

                

                    You can fuzzy search for courses by their name and description with the keywords parameter.
                    The keywords parameter is a string where the words are space-delimited.
                    This API will reject any string with more than 5 keywords (stopwords are not included).
                    Here is an example of a query using the keywords parameter:
                

                
GET /api?keywords=electric theories magnets

                

            

            

                
Excluding Data

                

                    You can exclude any unnecessary data using the excludes parameter.
                    The excludes parameter is a string where the unnecessary fields are space-delimited.
                    You can exclude all properties using *.
                    Here is an example of a query using the excludes parameter:
                

                
GET /api?subject=SPN&excludes=subject code units description

                

            

            

                
Pagination

                

                    You can limit the number of results with the limit parameter.
                    You can specify an offset in the data using the skip parameter.
                    The limit and skip parameters should be positive safe integers.
                    Pagination is possible when combining both of these parameters.
                    Here is an example of a query that makes use of pagination:
                

                
GET /api?subject=CHM&limit=10&skip=

                

            

            

                
Ordering Results

                

                    You can order the results using the sortBy parameter.
                    Reverse the order of results by using the reverseOrder parameter.
                    Here is an example of a query using these parameter:
                

                
GET /api?subject=CHM&limit=3&sortBy=units&reverseOrder
